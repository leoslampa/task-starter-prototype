# 一句话任务启动助手 MVP 草案

## 1. 一句话定位

帮助项目负责人在收到一句模糊任务后，快速生成一份可拿去确认的小报告。

核心方法：

> 先消化，再思考，慎提问。

## 2. MVP 场景

上级或需求方给出一句话任务，项目负责人需要在 30 分钟内形成：

- 初步理解
- 关键缺口
- 3-5 个最该问的问题
- 可留痕的确认话术

MVP 只解决“任务启动前怎么理解和发问”，不做完整项目管理。

## 3. 用户输入

用户先只填 3 项：

| 输入项 | 说明 | 示例 |
|---|---|---|
| 一句话任务 | 对方原话，用于后续留痕 | 下个月把新员工培训搞起来 |
| 任务是谁给的 | 判断沟通方式和向上确认必要性 | 上级 |
| 什么时候要反馈 | 判断报告颗粒度和问题优先级 | 明天上午例会前 |

可选补充：

- 已知背景
- 可能涉及的人或部门
- 当前最担心的问题

可选补充不放在主流程里，避免用户一开始就被表单劝退。

## 4. 系统输出

系统输出一份小报告，分为三块。

## 5. 报告结构

### 5.1 先消化

目的：先把能从原话里看出来的信息整理出来，避免一上来就乱问。

输出：

- 原始任务
- 能确定的信息
- 明显缺失的信息
- 可以先自行核查的信息

### 5.2 再思考

目的：把需求语言转成项目语言，形成项目负责人自己的初步判断。

输出：

- 项目负责人当前理解
- 可能的任务边界
- 最大偏航风险

说明：

- “项目负责人当前理解”不是最终结论，而是待确认的理解草案。
- “任务边界”用于说明本次任务可能包含什么，以及暂不默认包含什么。
- “最大偏航风险”用于提醒负责人哪些地方最容易“看起来合理但方向错了”。

### 5.3 慎提问

目的：帮助项目负责人少问、问准、问得让对方愿意回答。

系统生成 3-5 个最值得问的问题。

询问清单字段：

| 待确认事项 | 优先级 | 询问对象 | 建议问法 | 向上确认的必要性 |
|---|---|---|---|---|

字段说明：

- 待确认事项：要确认什么，以及为什么重要。
- 优先级：高、中、低。
- 询问对象：只写需要问的人，不写“自己查资料”。
- 建议问法：尽量使用确认式表达，而不是把空白问题直接抛给对方。
- 向上确认的必要性：必须、建议、可暂缓、无需。

## 6. 判断规则

重要度为高的问题，通常涉及高频重点决策。

凡是项目负责人不能自行决定，且判断错误会造成方向偏差、返工或责任争议的问题，都应优先考虑向上确认。

常见需要向上确认的问题包括：

- 目标：最终要达成什么结果。
- 交付物：最后要交什么。
- 时间：什么时候要完成或反馈。
- 任务边界：哪些做，哪些不默认做。
- 验收标准：怎样算完成。
- 资源承诺：是否需要人、钱、权限或跨部门支持。

## 7. 书面记录 Checklist

小报告最后给项目负责人 5 条留痕提醒：

| 检查项 | 是否完成 |
|---|---|
| 保留原始任务原话 |  |
| 写明项目负责人当前理解 |  |
| 确认目标和交付物 |  |
| 确认时间和任务边界 |  |
| 留下确认记录和时间 |  |

## 8. 示例

### 用户输入

| 输入项 | 内容 |
|---|---|
| 一句话任务 | 下个月把新员工培训搞起来 |
| 任务是谁给的 | 上级 |
| 什么时候要反馈 | 明天上午例会前 |

### 先消化

能确定的信息：

- 任务与新员工培训有关。
- 时间上至少要求下个月有所动作。
- 任务来自上级，需要注意目标和交付标准的确认。

明显缺失的信息：

- “搞起来”是指完成一场培训，还是建立长期培训机制。
- 最终希望看到什么交付物。
- 下个月需要完成到什么程度。

可以先自行核查的信息：

- 是否有历史培训方案。
- 近期新员工人数和岗位分布。
- 是否已有课程、讲师或制度要求。

### 再思考

项目负责人当前理解：

> 我目前理解，本任务可能是要求在下个月完成一版新员工培训的启动或落地，至少包括培训对象确认、课程安排、讲师协调和反馈记录。但“搞起来”具体是指完成首场培训，还是建立长期培训机制，还需要确认。

可能的任务边界：

- 先聚焦近期新员工入职培训。
- 暂不默认覆盖老员工复训。
- 暂不默认建设完整在线学习平台。
- 暂不默认制定全年培训体系。

最大偏航风险：

- 如果未确认“搞起来”的含义，可能把一次培训误做成长期体系，或把长期机制误做成一次活动。

### 慎提问

询问清单：

| 待确认事项 | 优先级 | 询问对象 | 建议问法 | 向上确认的必要性 |
|---|---|---|---|---|
| 确认本次任务最终希望达成的结果，避免把一次培训和长期机制混淆 | 高 | 上级/任务提出人 | 我先按“下个月完成首版新员工培训落地”来推进，您看这个理解是否准确？如果您更希望建立长期机制，我会调整方案。 | 必须 |
| 确认下个月需要完成到什么程度，避免排期和交付标准偏差 | 高 | 上级/任务提出人 | 这个任务是希望下个月完成首场培训，还是下个月拿出完整方案并启动实施？ | 必须 |
| 确认最终希望看到的交付物，避免交付形式不符合预期 | 高 | 上级/任务提出人 | 最终您更希望看到的是培训执行结果、培训方案文档，还是后续机制设计？ | 必须 |
| 确认培训对象范围，避免覆盖范围过大或遗漏重点人群 | 中 | 人力/相关部门负责人 | 我先确认一下近期新员工名单和岗位分布，方便判断课程重点和培训批次。 | 可暂缓 |
| 确认任务边界，避免范围扩大到全年培训体系或平台建设 | 高 | 上级/任务提出人 | 为保证下个月先落地，我建议这次先聚焦新员工入职培训，不扩展到全年培训体系和平台建设，您看可以吗？ | 建议 |

对上确认话术：

> 我先按以下理解推进：本次任务重点是下个月完成首版新员工培训落地，先覆盖近期入职新员工，形成培训方案、日程安排、讲师协调和反馈记录。为避免方向偏差，我想确认三点：第一，本次是完成首场培训，还是建立长期培训机制；第二，下个月需要完成到什么程度；第三，最终您希望看到的交付结果是执行记录、方案文档，还是机制设计。确认后我会整理一版简要推进方案。

## 9. 暂不做

MVP 阶段暂不做：

- 完整项目排期
- 甘特图
- 任务分派
- 执行进度跟踪
- 多轮审批
- 企业知识库自动检索

这些能力等“任务启动小报告”验证有效后再考虑。
